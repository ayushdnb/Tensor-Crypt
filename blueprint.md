# Blueprint

## Architecture

This branch is the single-brain-vmap release line. Its canonical operator path is a self-centric single-family runtime that preserves Tensor Crypt's UID-owned simulation substrate.

The canonical implementation root is `tensor_crypt/`. Root files and `engine/` or `viewer/` packages are compatibility surfaces.

## Split Doctrine

The single-brain-vmap line keeps experimental self-centric observations, branch-family enforcement, UID-owned PPO, binary-parented respawn, telemetry ledgers, checkpoint validation, catastrophe controls, and the pygame-ce viewer.

The five-family taxonomy remains only where it is still needed for compatibility, validation, telemetry, checkpoint metadata, and viewer surfaces. Normal public startup activates one branch family.

## Invariants

- Slots are storage; UIDs are identity.
- The branch family is checkpoint-visible and must match on restore.
- Experimental observation shapes are checkpoint and topology relevant.
- Vmap is an execution path, not parameter sharing.
- Runtime validation should fail unsupported config values loudly.

## Branch Policy

Branch: `release/single-brain-vmap`

Version target: `0.9.0` while the config field names and torch.func dependency remain explicitly experimental.

Tag target: `single-brain-vmap/v0.9.0` only after successful validation.
